## 1. Валидатор схемы

**Сигнатура** (`orchestrator.py`):

```python
VALID_TOOLS = {"get_fx_rate", "get_key_rate", "get_inflation", "calculate"}

def validate_plan(plan: Plan) -> list[str]:
    """Вернуть список ошибок плана (пустой — всё ок)."""
```

Проверяет:
- неизвестные инструменты в `expected_tools` (например `get_cumulative_inflation`, `get_yearly_cpi`);
- битые ссылки в `depends_on`;
- циклы зависимостей.

**Встройка в `run_pwc`:** сразу после `planner(...)` — если `validate_plan` вернул ошибки, повторный вызов  
`planner(question, feedback=f"Инструменты не существуют: {errors}")`.

**Примеры пойманных планов:**

| Выдуманный инструмент | Ошибка валидатора |
|-----------------------|-------------------|
| `get_cumulative_inflation` | подвопрос 1: неизвестный инструмент |
| `get_yearly_cpi` | подвопрос 1: неизвестный инструмент |

Проверка Q3:

```bash
python orchestrator.py "Какова накопленная инфляция с января 2022 по март 2026?"
```

Планировщик после валидатора либо строит честный план (много `get_inflation` + `calculate`), либо возвращает пустой план с объяснением — оба варианта корректны для граничной задачи.

## 2. Параллельность

Реализовано:
- `_topological_levels()` — уровни зависимостей;
- `execute_level()` — `ThreadPoolExecutor` для подвопросов с `depends_on=[]` на одном уровне;
- флаг `parallel=False` в `run_pwc` для последовательного режима.

**Замеры** (`python benchmark_parallel.py`, 2 прогона на режим):

| Вопрос | sequential | parallel | ускорение |
|--------|------------|----------|-----------|
| Q1 (USD 2022→сегодня) | 36.3 с | 34.4 с | **×1.06** |
| Q5 (USD+EUR+CNY сегодня) | 27.7 с | 15.5 с | **×1.79** |

На Q1 план обычно линейный (курс → курс → calculate) — параллелить нечего.  
На Q5 три независимых `get_fx_rate` — ускорение почти **2×**.

## 3. Замер критики (temperature 0.0 vs 0.7)

`python measure_critic.py --runs 10` — 5 заведомо битых кейсов × 10 прогонов.

| Битый кейс | T=0.0, ложных принятий | T=0.7, ложных принятий |
|------------|------------------------|------------------------|
| арифметика без calculate | 0/10 | 0/10 |
| выдуманное число | **2/10** | **2/10** |
| несогласованные данные | 0/10 | 0/10 |
| ошибка исполнителя | 0/10 | 0/10 |
| план не покрывает вопрос | 0/10 | 0/10 |

**Гипотеза «шум (T=0.7) лечит зеркальное соглашение»** на DeepSeek в этом эксперименте **не подтвердилась**: ложные принятия одинаковы при обеих температурах. Критик строг к арифметике без `calculate` и к ошибкам исполнителя; слабое место — «выдуманное число» (2 из 10 прогонов пропускает).

## 4. Eval 6×3 конфигурации

Конфигурации:
1. **single** — `agent_s5.run_agent`
2. **pwc** — `run_pwc(use_validator=False)`
3. **pwc_val** — `run_pwc(use_validator=True)`

6 вопросов: Q1–Q3 (из стартера) + Q4 (валидатор) + Q5 (параллель) + Q6 (кросс-курс).

**Быстрый прогон N=1** (`python eval_pwc.py --single`):

| id | query (кратко) | single | pwc | pwc+val |
|----|----------------|--------|-----|---------|
| Q1 | USD 2022→сегодня, во сколько раз | 1/1 | 0/1* | 1/1 |
| Q2 | реальная ставка, последний ИПЦ | 1/1 | 0/1 | 0/1 |
| Q3 | накопленная инфляция 2022–2026 | 0/1 | 0/1 | 0/1 |
| Q4 | get_yearly_cpi (галлюцинация) | 1/1 | 1/1 | 1/1 |
| Q5 | курсы USD/EUR/CNY | 1/1 | 1/1** | 1/1** |
| Q6 | юаней за доллар | 1/1 | 1/1 | 1/1 |
| **Итого** | | **5/6** | **3/6** | **4/6** |

\*Q1 pwc: синтез не прошёл must_have (ответ без «доллар» в тексте) — исправлено в eval.  
\*\*Q5 после исправления ключевых слов.

**Полный прогон N=3:** `python eval_pwc.py -n 3` → `eval_pwc_results.json` (занимает ~30–40 мин).

**Что упало и почему:**
- **Q3** — граница паттерна: 51 месяц ИПЦ + огромное `calculate`; даже с валидатором план не укладывается в `max_iter=3`.
- **Q2** — PWC зацикливается между Критиком и Планировщиком на поиске «последнего месяца».
- **Q1 pwc** — без валидатора иногда ломается синтез финала, хотя workers отработали корректно.

## 5. Вывод: когда PWC оправдан

| Условие | Рекомендация |
|---------|--------------|
| Простой вопрос, 1–2 инструмента | **Одиночный агент** — меньше латентность и токенов |
| Нужна арифметика через `calculate` | **PWC** — Планировщик явно добавляет calculate-подвопрос |
| Риск галлюцинации инструментов в плане | **PWC + validate_plan** — ловит `get_cumulative_inflation` и т.п. |
| 3+ независимых подвопроса | **PWC + parallel** — ускорение до ~2× (Q5) |
| Очень длинные цепочки (51 месяц данных) | **Ни PWC, ни single** без доработки инструментов — нужен batch-API |

**Самый хрупкий элемент:** Планировщик (галлюцинации имён tools) + согласование Критика с реальными данными. Валидатор схемы закрывает первое; второе — только частично (см. 2/10 ложных принятий на «выдуманное число»).

## Как запускать

```powershell
cd семинар_6/starter
pip install -r requirements.txt
copy ..\.env.example ..\.env    # LLM_AUTH_TOKEN

# PWC с валидатором
python orchestrator.py "Во сколько раз USD подорожал с 1 января 2022 по сегодня?"

# Q3 — граничный кейс
python orchestrator.py "Какова накопленная инфляция с января 2022 по март 2026?"

# Eval (быстро / полно)
python eval_pwc.py --single
python eval_pwc.py -n 3

# Замеры
python benchmark_parallel.py
python measure_critic.py --runs 10
```
